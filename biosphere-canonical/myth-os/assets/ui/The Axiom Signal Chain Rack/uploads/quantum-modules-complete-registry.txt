Quantum Modules — Complete Registry
16 modules. 4 departments. 4 modules each. Every module fully Crested. Every module has exactly 16 Standard Components organized into 4 layers of 4.
Genesis, Core, and Vault are infrastructure — not modules. Do not add, rename, or move any module.

Sub-Module Layer — Level 4
Each Standard Component contains exactly 16 Sub-Modules. Full data lives in the 16 companion JSON files in this skill folder — one per module. Load the relevant JSON when building a module's node graph, writing TypeScript for its implementation, or generating component specs.
JSON files: 01-terrain.json · 02-environment.json · 03-architect.json · 04-lighting.json · 05-modeling.json · 06-choreography.json · 07-behavior.json · 08-society.json · 09-sequencer.json · 10-story.json · 11-memory.json · 12-sound.json · 13-logic.json · 14-simulation.json · 15-forge.json · 16-network.json
Per-file structure: module identity → 16 standard_components → 16 sub_modules each → 256 addressable nodes per module. Sub-modules pending for modules 03–16 are marked "status": "pending".
UIX wire type: display-layer output only — visual feedback and user interaction events. Never crosses into simulation logic.

Hardware Highlight:
Skeleton/Rigging Engine (RIG-18): Mapped to PWM servo controllers. If you create a "creature" in the Modeling module, these servos can physically replicate the pose of your digital character on a physical armature.
Physical Manifestation (FAB-20): This component talks to your 3D printer. You can "Materialize" a digital object directly out of the Modeling module. By tweaking the Infill Density knob on your MIDI controller, you are literally changing the internal structural strength of the physical object being printed.
