Title: Eidolon Synthesis Rack

Description:
A modular, real-time simulation control interface designed for the Fyrox Engine, conceptualized as a rack of professional, haunted synthesizers for orchestrating reality. Each module translates a fundamental cosmic function—from quantum entanglement to narrative causality—into a tangible, performable instrument.

1. **Core Philosophy:** The Eidolon Synthesis Rack embodies the principle of 'Performative Ontology'—the act of creating and modulating existence through tactile, real-time interaction. It rejects abstract code and menus in favor of a musician's cockpit, where the user, as a World Weaver and Flux Orchestrator, patches, tweaks, and sequences the very laws of the simulation. The 'Celestial Haunted Sonic Waves Prism' theme is manifest in every interaction; turning a knob fractures light into a prism, patching a cable creates a singing, spectral tendril, and the entire console hums with the haunted, resonant frequency of the active world-state.

2. **Layout & Structure:** The interface is constructed as a primary Fyrox scene containing a single, full-screen UI root. The core element is a vertically scrollable `ScrollViewer` widget, which serves as the 'Rack'. This rack is rendered using 3D meshes to give it a physical, dark iridescent metallic presence against a dynamic celestial nebula background. The rack houses 'Instruments' (e.g., 'Mythos Cartographer', 'Quantum Simulation Director'), which are complex user-defined widgets built from `Grid` layouts. Each Instrument panel is populated by smaller, modular 'Eurorack' units, each a discrete custom Fyrox widget representing a specific function like a 'Tornado' joystick controller or a 'Causality' sequencer. Double-clicking any Eurorack module transitions the view to its underlying logic, a 2D `Canvas`-based node graph editor where raw axiomatic connections are made.

3. **Key Features & Interactions:**
    - **Somatic Signal Patching:** Users physically connect modules using glowing, bioluminescent 'cables' drawn directly onto the UI canvas via Fyrox's `DrawingContext`. For example, a user might patch the LFO (Low-Frequency Oscillator) output from a 'Tidal Choreographer' module into the 'Probability Amplitude' input on a 'Probability Seer' module. This connection is more than visual; in the Rust backend, it pipes the rhythmic data from one system's logic loop directly into the parameters of another, causing probabilistic events in the simulation to pulse in time with resource tides. The cables hum and pulse with light corresponding to the data they carry.
    - **Axiom Step Sequencer:** A dedicated Instrument module for the 'Axiom Carver' role. It presents as a classic 16-step sequencer, but instead of drum sounds, each lane corresponds to a fundamental law of the simulation (e.g., Gravity, Temporal Flow, Logical Consistency). Activating a step on a lane 'triggers' that law for a beat, allowing the user to create rhythmic, temporary pockets of altered physics or looping causalities. This provides an intuitive, musical interface for scripting complex, time-based systemic changes.
    - **Resonance Harmonics Engine:** This module, representing the 'Resonance Sealer' and 'Harmonic Resonator' roles, features a bank of sliders styled after a drawbar organ. Each slider controls the amplitude of a specific 'harmonic' of the world's core resonant frequency. Pushing these sliders not only alters the ambient sonic environment but also influences emergent behaviors system-wide. For instance, boosting the '7th harmonic' might subtly encourage paradoxical entities ('Chimera') to stabilize, while dampening the '2nd harmonic' could suppress unwanted emergent behavior cultivated by the 'Emergence Botanist'.

4. **Data Visualization:** Data is presented synesthetically, prioritizing sonic and visual feedback over numerical readouts. 
    - **The Entanglement Orrery:** Within the 'Entanglement Architect' module, a small, embedded 3D viewport renders quantum-entangled pairs as miniature, orbiting celestial bodies. Their color, speed, and orbital decay visualize their superposition and coherence, with a probability collapse event triggering a brilliant, prismatic flare and a sharp, crystalline chime. This is a dedicated Fyrox scene rendered to a texture and displayed on a UI `Image` widget.
    - **Flux Spectrogram:** A central oscilloscope-like visualization, built as a custom procedural mesh in the Fyrox scene, represents the total data flow managed by the 'Flux Orchestrator'. It renders data streams as shimmering, interwoven sonic waves. System stability is felt as a consonant chord, while data bottlenecks or errors manifest as jarring visual noise and audible dissonance, guiding the operator to the source of the problem instinctively.

5. **Ambiance & Feel:** The experience is dark, focused, and intensely creative. It feels less like operating software and more like being physically plugged into the simulation's nervous system from the deck of a mythic vessel adrift in a cosmic void. The tightly-packed, iridescent controls with their bioluminescent indicators demand precision and reward mastery. The 'Haunted' feel is ever-present, with ghostly light trails following control movements and the faint, mnemonic echoes of past causal chains whispering in the ambient soundscape. It is a professional and deeply immersive environment for the god-like act of sculpting worlds.
